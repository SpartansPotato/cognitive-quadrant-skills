# 兼容设计说明

本仓库不为每个平台复制一套不同的核心指令，而是每项能力只维护一个 Skill 目录。

- WorkBuddy 读取 `SKILL.md`、`manifest.yaml`、参考文档和模板资源；
- Codex 读取 `SKILL.md`、参考文档、资源，以及可选的 `agents/openai.yaml`；
- Claude Code 读取 `SKILL.md`、参考文档、资源和示例，目录名称同时作为斜杠命令名称。

某个平台不认识的元数据文件保持静态，不会参与核心流程。Skill 在调用联网、文件、图片、表格或文档生成能力前，必须先判断宿主当前是否真正提供这些工具。

## “兼容”的边界

这里的“兼容”表示工具包遵循平台公开的目录与元数据规范，并且没有主动锁定某个平台；不代表已经对所有模型、客户端版本和权限组合完成穷尽测试。
